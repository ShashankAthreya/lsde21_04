Community Detection

Community detection is a method for finding groups within complex systems represented on a graph. In contrast to more traditional decomposition methods which seek a strict block diagonal or block triangular structure, community detection methods find subnetworks with statistically significantly more links between nodes in the same group than nodes in different groups (Girvan and Newman, 2002).

We have used Gephi to identify these communities.

Nodes

We are mapping all address associated to a single person as a single node mainly to reduce the size of the graph.

Edges

Each edge is a transaction between two people and/or organisations. Repeatitive edges are noted and then collapsed into a single edge.

Relationships

We observe repeated transactions between certain nodes and analysing these edges lead us to identifying communities.

Algorithms

We use the Fast Unfolding of Communities algorithm for detecting communities within the generated graphs.

Initial Graph size.

When we talk about the size of the graph, we are strictly speaking about the number of edges of the graph. There are roughly 230000 transactions in a single blockfile which leads to a gigantic graph.

Since this is quite impossible to comprehend, we will have to reduce the size of the nodes and then the edges. We did that by merging addresses that belonged to the same owner; even if the owner had multiple wallets, it would still point only to him.

Bitcoin Transaction Analysis(Data Exploration)

A single block consists of a header and transaction data. The parameters are changed on the amount of transactions and computational power of the miner, to keep producing a new block roughly every 10 minutes. We can see that there is a lot of difference in the amount of blocks per file in the graph. The first include more blocks than later files. This is because a block size grows as the amount of transactions per 10 minutes grows. To keep the size of the files roughly the same, there need to be less blocks in later files.

Total Blocks

We have a total of 2742 blkxxxxx.dat files.

Encryptions

All these blocks are in binary and hashed by SHA-256.

Transactions

Each block-line contains the transactions of the last 10 minutes.

The Bitcoin Block Structure is heavily nested,

A bitcoin block is limited to 128MBs. Because a block is produced at a stable rate, there is a maximum amount of Bitcoin transactions per unit of time. The maximum is estimated at about 4.6 transactions per second. We can see this amount is reached in figure 3. The Transactions per block through the years.

We can observe a gradual decrease in the number of transactions per block in the recent years. Perhaps this has something to do with the price of a bitcoin or the presence of other cryptocurrencies; we can not state the reason with certainity.

Similarities to the Barabasi-Albert model.

The Barabasi-Albert Model depicts a graph of Network Systems. We observe the network hubs are the nodes with highest number of edges.

We can see the similarities with the slope of the Barabasi-Albert model and the User-Transaction distribution. This leads us to beleive that there are certain hubs within the Bitcoin graphs and analysis of these hubs are interesting.

These hubs can be the platforms which allow coin trade or they can be great influencers. Regardless, we can assume they play a dominant role in the community as they hold the highest modularity score.
